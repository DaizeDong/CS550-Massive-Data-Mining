# CS550 推荐系统实验项目总结

## 一、项目概述

在 MovieLens-1M 数据集上实现并对比 **7 个推荐模型**，涵盖两个任务：
- **评分预测**（Rating Prediction）：预测用户对物品的 1-5 评分
- **Top-10 推荐**（Top-N Recommendation）：为用户推荐最可能感兴趣的 10 个物品

所有模型从零实现（NumPy + PyTorch），不使用任何推荐系统库。

---

## 二、数据集与预处理

### 数据集：MovieLens-1M
| 属性 | 数值 |
|------|------|
| 原始评分数 | 1,000,209 |
| 用户数 | 6,040 |
| 原始物品数 | 3,706 |
| 评分范围 | 1-5 的整数 |
| 每个用户最少评分数 | 20 |

### 预处理步骤
1. **物品过滤**：删除评分数少于 5 的物品，剩余 3,416 个物品
2. **ID 重映射**：将用户和物品 ID 映射为连续整数 (0 ~ N-1)，方便后续建立 Embedding 表
3. **数据划分**：对每个用户，随机抽取 20% 的评分作为测试集（seed=42）
   - 训练集：799,721 条
   - 测试集：199,890 条
   - 稀疏度：95.16%
   - 保证每个用户在训练集和测试集中至少有 1 条记录

## 三、7 个模型详解

### 3.1 GlobalMean（全局均值）

最简单的基线。对所有用户-物品对，预测值都等于训练集的全局平均评分 $\mu$。

$$\hat{r}_{ui} = \mu$$

没有可学习参数，仅用于衡量其他模型的改进幅度。

---

### 3.2 BiasOnly（偏置模型）

在全局均值的基础上加入用户偏置 $b_u$ 和物品偏置 $b_i$：

$$\hat{r}_{ui} = \mu + b_u + b_i$$

- 训练方式：交替的闭式 Ridge 更新（不是梯度下降）
- 正则化系数 $\lambda = 5$，迭代 15 轮
- 直觉：有的用户评分偏高，有的物品评分偏低，偏置项可以捕捉这种倾向

---

### 3.3 ItemKNN（基于物品的协同过滤）

核心思想：如果用户喜欢物品 A，且物品 B 跟 A 很相似，那用户也可能喜欢 B。

**相似度计算**：使用带收缩因子的调整余弦相似度

$$s_{ij} = \frac{\sum_{u \in \mathcal{U}_{ij}} \tilde{r}_{ui} \tilde{r}_{uj}}{\|\tilde{r}_{\cdot i}\| \|\tilde{r}_{\cdot j}\|} \cdot \frac{|\mathcal{U}_{ij}|}{|\mathcal{U}_{ij}| + \alpha}$$

- $\tilde{r}_{ui} = r_{ui} - \bar{r}_u$：去掉用户均值后的评分
- $\alpha = 100$：收缩因子，防止共同评分用户少的物品对获得过高相似度

**预测公式**：取每个物品最相似的 $k=50$ 个邻居：

$$\hat{r}_{ui} = \bar{r}_u + \frac{\sum_{j \in N_k(i) \cap \mathcal{I}_u} s_{ij} \cdot \tilde{r}_{uj}}{\sum_{j \in N_k(i) \cap \mathcal{I}_u} |s_{ij}|}$$

---

### 3.4 UserKNN（基于用户的协同过滤）

与 ItemKNN 原理相同，但计算的是用户之间的相似度。核心思想：如果用户 A 和用户 B 的历史评分很相似，那 A 喜欢的物品 B 也可能喜欢。

参数与 ItemKNN 一样：$k=50$, $\alpha=100$。

---

### 3.5 MF（带偏置的矩阵分解）

将每个用户和物品分别表示为一个 $d$ 维向量（latent factor），通过向量内积来预测评分：

$$\hat{r}_{ui} = \mu + b_u + b_i + p_u^\top q_i$$

| 参数 | 值 |
|------|-----|
| 隐因子维度 $d$ | 64 |
| 训练轮数 | 20 |
| 优化器 | Adam |
| 学习率 | 0.01 |
| L2 正则 | $10^{-4}$ |
| 批大小 | 8192 |
| 损失函数 | MSE |

使用 PyTorch 实现，支持 GPU 加速。预测值裁剪到 [1, 5]。

---

### 3.6 NeuMF（神经协同过滤）

NeuMF 包含两个并行分支：

1. **GMF 分支**：$p_u^g \odot q_i^g$（逐元素乘积，维度 32）
2. **MLP 分支**：将 $[p_u^m \| q_i^m]$ 拼接后通过三层全连接 (64 → 32 → 16)，每层后接 ReLU 和 Dropout(0.1)

两个分支的输出拼接后通过一个线性层输出最终预测：

$$\hat{r}_{ui} = \mu + \text{Linear}([g \| h])$$

| 参数 | 值 |
|------|-----|
| GMF 维度 | 32 |
| MLP 维度 | 32 |
| MLP 隐层 | (64, 32, 16) |
| Dropout | 0.1 |
| 学习率 | 0.001 |
| 损失函数 | MSE |

注意：原始 NeuMF 用二分类交叉熵（隐式反馈），这里改用 MSE（显式评分任务）。

---

### 3.7 BPR-MF（贝叶斯个性化排序）

BPR-MF 的评分函数和 MF 一样（$p_u^\top q_i + b_i$），但使用不同的损失函数。

**训练方式**：不预测具体评分值，而是学习排序。对每个用户，采样一个正样本物品 $i$（用户交互过的）和一个负样本 $j$（未交互的），训练目标是让正样本得分高于负样本：

$$\mathcal{L}_{\text{BPR}} = -\sum_{(u,i,j)} \log \sigma(\hat{x}_{ui} - \hat{x}_{uj}) + \lambda \|\Theta\|^2$$

| 参数 | 值 |
|------|-----|
| 隐因子维度 | 64 |
| 训练轮数 | 20 |
| 批大小 | 4096 |
| 学习率 | 0.01 |
| L2 正则 | $10^{-5}$ |

负采样方式：均匀采样，如果采到正样本则重新采（最多重试 10 次）。

BPR-MF 输出的分数不是 1-5 评分，所以不计算 MAE/RMSE。

---

### 3.8 LightGCN（轻量图卷积网络）

在用户-物品二部图上做图卷积，让每个节点聚合邻居信息。与标准 GCN 不同，LightGCN 去掉了特征变换和非线性激活，只保留加权平均传播：

$$\mathbf{E}^{(\ell)} = \hat{\mathbf{A}} \mathbf{E}^{(\ell-1)}$$

其中 $\hat{\mathbf{A}} = D^{-1/2} A D^{-1/2}$ 是对称归一化的邻接矩阵。最终 embedding 是所有层输出的平均：

$$\mathbf{E}^{*} = \frac{1}{L+1} \sum_{\ell=0}^{L} \mathbf{E}^{(\ell)}$$

| 参数 | 值 |
|------|-----|
| 隐因子维度 | 64 |
| 图卷积层数 $L$ | 3 |
| 训练轮数 | 15 (初始) / 20 (匹配实验) |
| 批大小 | 16384 (初始) / 4096 (匹配实验) |
| 学习率 | 0.001 |
| L2 正则 | $10^{-4}$（仅对初始 embedding） |
| 损失函数 | BPR Loss |

LightGCN 跑了两次：
1. **初始运行**：15 epochs, batch=16384，约 720 个梯度步
2. **匹配运行**：20 epochs, batch=4096，约 3900 个梯度步（与 BPR-MF 一致）

目的是排除 "LightGCN 只是训练不够" 的可能性。

---

## 四、评估指标

### 评分预测指标
- **MAE**（平均绝对误差）：$\frac{1}{|\mathcal{D}|}\sum |r - \hat{r}|$
- **RMSE**（均方根误差）：$\sqrt{\frac{1}{|\mathcal{D}|}\sum (r - \hat{r})^2}$

预测值裁剪到 [1, 5] 后计算。

### Top-N 排序指标（K=10）

对每个用户：对所有物品打分 → 去掉训练集中已有的 → 取 Top-10 → 与测试集对比。

- **Precision@K**：Top-K 中命中测试集的比例
- **Recall@K**：命中数 / 用户测试集总数
- **F1@K**：Precision 和 Recall 的调和平均
- **NDCG@K**：使用二值相关性（在测试集中=1，否则=0），衡量排序质量

所有指标在所有用户上取宏平均。

---

## 五、实验结果

### 主要结果

| 模型 | MAE ↓ | RMSE ↓ | P@10 ↑ | R@10 ↑ | NDCG@10 ↑ | 训练时间 |
|------|-------|--------|--------|--------|-----------|---------|
| GlobalMean | 0.933 | 1.116 | 0.009 | 0.002 | 0.009 | 0s |
| BiasOnly | 0.716 | 0.905 | 0.053 | 0.020 | 0.049 | 0.4s |
| ItemKNN | 0.714 | 0.931 | 0.013 | 0.006 | 0.014 | 1.8s |
| UserKNN | 0.707 | 0.918 | 0.012 | 0.004 | 0.013 | 4.0s |
| MF | 0.704 | 0.887 | 0.112 | 0.038 | 0.125 | 33.5s |
| NeuMF | 0.704 | 0.904 | 0.090 | 0.031 | 0.098 | 87.5s |
| BPR-MF | N/A | N/A | **0.286** | **0.107** | **0.320** | 49.0s |
| LightGCN | N/A | N/A | 0.182 | 0.067 | 0.204 | 2270.8s |

### 关键发现

**1. BPR 损失函数的优势**

BPR-MF 和 MF 用的是同一个评分函数 $p_u^\top q_i + b_i$，唯一区别是损失函数（BPR vs MSE）。结果 BPR-MF 的 P@10 是 MF 的 2.55 倍（0.286 vs 0.112），NDCG@10 是 2.57 倍（0.320 vs 0.125）。原因是 MSE 训练目标是拟合每个评分值（包括 2 分、3 分），对排序没有直接帮助；而 BPR 直接优化正负样本之间的相对顺序，正好是排序指标衡量的目标。

**2. kNN 模型：评分准但排序差**

UserKNN 和 ItemKNN 的 MAE 不错（0.707 和 0.714），但排序指标很低（NDCG 只有 0.013 和 0.014）。原因是 kNN 对用户没评分过的物品，预测值会退化到用户均值附近，分数区分度不够。另外，kNN 用来预测的主要是用户已经评过分的邻居物品，而这些物品在 Top-N 评估时已经被排除了。

**3. LightGCN 不如 BPR-MF**

LightGCN 的所有排序指标都低于 BPR-MF（P@10: 0.183 vs 0.286），而且训练时间长了 46 倍。匹配梯度步数后（20 epochs, batch=4096），结果几乎没变化，说明不是训练不够的问题。可能的原因：
- $L=3$ 层可能不是最优的，$L=2$ 可能更好（没有做超参搜索）
- ML-1M 每个用户至少有 20 个评分，直接的因子模型已经有足够信号，图卷积引入的多跳邻居信息帮助不大
- 正则化参数没有调优

**4. NeuMF 没有比 MF 更好**

NeuMF 的排序指标反而略低于 MF（NDCG: 0.098 vs 0.125），额外的神经网络层在这个数据集上没有带来提升。

## 六、公平性分析

除了准确性，还分析了模型是否过度集中推荐热门物品。

| 模型 | Coverage@10 ↑ | Gini ↓ | Long-tail Share ↑ |
|------|--------------|--------|-------------------|
| GlobalMean | 0.103 | 0.987 | 0.873 |
| BiasOnly | 0.095 | 0.995 | 0.260 |
| ItemKNN | **0.821** | **0.690** | 0.557 |
| UserKNN | 0.757 | 0.749 | **0.870** |
| MF | 0.052 | 0.994 | 0.015 |
| NeuMF | 0.480 | 0.876 | 0.257 |
| BPR-MF | 0.204 | 0.975 | 0.005 |
| LightGCN | 0.022 | 0.995 | 0.000 |

三个指标的含义：
- **Coverage**：有多少比例的物品至少出现在了某个用户的 Top-10 中。越高说明推荐越多样
- **Gini 系数**：物品曝光次数的不平等程度。0 表示完全均匀，1 表示极度不均
- **Long-tail Share**：推荐列表中来自长尾物品（训练集中热度排名后 80%）的比例

发现：排序最好的 BPR-MF 和 LightGCN 公平性最差（LightGCN 的 Long-tail Share 为 0）。反过来，kNN 模型公平性好但排序差。准确性和公平性在这个数据集上表现为此消彼长的关系。

```
  data_utils.py        数据加载、过滤、ID重映射、80/20划分

  models_classical.py  GlobalMean, BiasOnly, ItemKNN, UserKNN

```
