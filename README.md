# CS 550 course project -- seven recommenders on MovieLens-1M

Option 1 (Recommender System) from the CS 550 project description.
Baselines through LightGCN on the same per-user 80/20 split with matching
evaluation, plus an optional fairness audit. Everything written from
NumPy/PyTorch primitives; no recommender library is imported.

## Submission contents

Per the project description, this submission is a single zip named
`dd1376_jw2070.zip` (the two authors' NetIDs joined by an underscore),
containing one folder `dd1376_jw2070/` with:

1. **Code**               -- `src/`, `demo/`, `run_all.sh`, `make_submission.sh`
2. **Report**             -- `report/report.pdf`, written with the ACM sigconf LaTeX template suggested in the project description
3. **Presentation slides**-- `slides/slides.pdf`

Everything above is produced by `bash run_all.sh` (end-to-end training,
evaluation, fairness audit, LaTeX recompile) and packaged by
`bash make_submission.sh dd1376 jw2070`.

## Reproduce

```bash
pip install numpy pandas scipy scikit-learn torch tqdm matplotlib flask
bash run_all.sh                              # full pipeline, ~15 min CPU
python -m demo.app                           # demo -> http://localhost:5000
```

`results/metrics.csv` and `results/fairness.csv` are the source of truth
for every number in the report; the `.tex` tables under `report/` are
generated from them by `src/make_artifacts.py`.

## Notes on design choices

The split is per-user uniform-random, as the spec requires. For ML-1M
(20-core on users) this leaves every user with $\ge 1$ training and
$\ge 1$ test row, so no special casing is needed.

BPR-MF and LightGCN produce uncalibrated scores, not 1--5 ratings. Their
MAE/RMSE cells are reported as N/A rather than as misleading scalars.
This is explained in both the report (Section 4) and the slides.

LightGCN is evaluated twice: a short run (15 epochs, batch 16{,}384) and a
matched-budget run (20 epochs, batch 4{,}096, same gradient-step count as
BPR-MF). Both are stored; the matched run is the one cited in the report.

The pickled kNN checkpoints strip a cached score matrix (~82 MB) via
`__getstate__`; it is rebuilt on first use after load.

## No external recommender libraries

Per the project policy, no packaged recommender algorithm is imported.
The third-party dependencies are only `numpy`, `pandas`, `scipy`,
`scikit-learn`, `torch` (used for `nn.Embedding`, `nn.Linear`, and
`optim.Adam` as building blocks), `tqdm`, `matplotlib`, and `flask`.

## What is not done

- Cold-start serving (models are transductive).
- Timestamps are unused.
- Single seed per model, no bootstrap confidence intervals.
- LightGCN layer count is fixed at $L = 3$; a sweep is the obvious next
  thing to try and is the most likely explanation for the BPR-MF gap.
